# 每日差异化综述与公众号草稿实施方案

## 1. 背景判断

同一主题每天检索时，文献高度重复是正常现象。尤其是土木、材料、器件可靠性等细分方向，真实新增高质量论文通常不是“每日级”，而是“周/月级”。因此系统不应把日报定义为“每天必须发现大量新论文”，而应定义为：

- 有新增时：写“今日新增证据 + 趋势变化”。
- 新增不足时：写“滚动观察 + 历史证据再组织 + 子议题深挖”。
- 无高质量新增时：明确说明“今日无足够新增高质量文献”，并输出短日报、监测摘要或专题回顾，而不是硬凑综述。

## 2. 当前检索能力评估

当前项目的检索能力属于“可用的多源工程检索”，但还不是“稳定的系统综述级检索”。

优势：

- 已支持 Hybrid：Paper Search 多源 + Sciverse 补充。
- Paper Search 已覆盖 Semantic Scholar、OpenAlex、Crossref、Europe PMC、HAL、BASE、CORE、Unpaywall。
- 已做 DOI/title/uniqueId 去重。
- 已有 relevanceScore、qualityScore、evidenceScore。
- Sciverse 可参与检索补充，并尝试 docId/fulltext snippet。
- 支持 LLM 动态扩展检索词，比固定关键词更适合泛化主题。

主要短板：

- 公开多源 API 的摘要、被引量、期刊字段不稳定，字段匹配需要持续清洗。
- 中文小众主题依赖 LLM 翻译和扩展，可能出现泛化误召回。
- Sciverse 正文片段依赖 docId 命中，不能保证每篇 DOI 都能反查到正文。
- 高质量新增文献不一定每天存在；强行按日生成“完整综述”会造成重复。
- 目前更像“自动证据雷达 + 初筛综述器”，还不是严格 PRISMA/Systematic Review 工具。

结论：

当前检索能力适合做“每日/每周前沿监测、滚动综述、证据雷达、公众号科普转写”。如果用于严肃学术投稿级系统综述，需要加入人工审查、检索式锁定、数据库导出、纳排标准和可复现实验日志。

## 3. 核心策略：日报改成 Daily Delta

新增一个 Daily Delta Engine，生成日报前先判断当天是否有足够新增证据。

### 3.1 文献持久化索引

新增文件或数据库表：

- `paper_index.jsonl`
- `topic_daily_state.json`
- `daily_delta_runs.jsonl`

每篇文献保存：

- `doi`
- `titleHash`
- `uniqueId`
- `source`
- `topicId`
- `firstSeenAt`
- `lastSeenAt`
- `lastUsedAt`
- `usedCount`
- `relevanceScore`
- `qualityScore`
- `evidenceScore`
- `noveltyScore`
- `abstractHash`

### 3.2 新颖度评分

新增 `noveltyScore`：

```text
noveltyScore =
  首次出现分
  + 近年/近期发表分
  + 与近 N 期文献低重复分
  + 高质量来源分
  + DOI/摘要/全文证据完整度分
  - 最近使用惩罚
```

建议阈值：

- `noveltyScore >= 70`：今日新增重点文献。
- `40 <= noveltyScore < 70`：可作为补充证据。
- `< 40`：仅进入历史背景，不作为今日重点。

### 3.3 三种日报模式

系统自动选择：

1. `fresh_daily`
   新增高质量文献足够，例如不少于 10 篇或不少于目标数量 30%。

2. `delta_brief`
   新增文献较少，但有明确变化。输出短日报，强调“较上一期变化”。

3. `topic_deep_dive`
   新增不足。由 LLM 在主标题下选择一个子议题，基于历史高质量证据做专题深挖。

4. `no_significant_update`
   新增不足且不适合深挖。输出监测说明，不硬写完整综述。

## 4. 主标题与每日小标题

建议采用：

```text
固定主标题 + 每日动态小标题 + 与上一期对比
```

示例：

```text
主标题：土木工程智能结构设计前沿日报
小标题：今日聚焦：从生成式结构优化到数字孪生运维的证据更新
```

小标题生成输入：

- 主题配置
- 今日新增文献标题与摘要
- 最近 3-7 期小标题
- 最近 3-7 期关键词
- 管理员配置的子议题池

约束：

- 小标题不能与近 7 期高度相似。
- 不夸大“突破”“颠覆”。
- 新增不足时标题应诚实，例如“本周证据回顾”或“专题观察”。

## 5. 检索流程改造

### 5.1 当前流程

```text
主题 -> LLM 扩展检索词 -> Paper Search/Sciverse 检索 -> 去重 -> 质量排序 -> 综述
```

### 5.2 目标流程

```text
主题配置
  -> 读取历史索引和最近日报
  -> LLM 生成今日检索计划
  -> 多源检索
  -> DOI/title/uniqueId 去重
  -> 字段标准化
  -> 相关性/质量/证据/新颖度打分
  -> 判断日报模式
  -> 生成小标题与差异摘要
  -> 写日报
  -> 保存历史索引
  -> 可选生成公众号草稿
```

### 5.3 检索计划

每天不是只用同一个 query，而是生成：

- `coreQueries`：主主题检索。
- `freshQueries`：近 1-3 个月新文献检索。
- `subtopicQueries`：今日子议题。
- `highImpactQueries`：高被引/综述/关键期刊方向。
- `negativeTerms`：避免跑偏的排除词。

管理员可配置：

- 最小新增文献数。
- 最大重复文献比例。
- 子议题池。
- 子议题轮转周期。
- 新增不足时是否允许专题深挖。
- 是否允许生成“无显著更新”短报。

## 6. 公众号自动草稿方案

优先实现“自动生成草稿”，不直接自动群发。

原因：

- 个人/订阅号/服务号的 API 权限不同。
- 自动发布接口受账号类型、认证、IP 白名单、接口权限影响。
- 草稿模式可控，失败风险低。

### 6.1 公众号内容生成

日报生成后，再生成公众号版：

- 标题
- 小标题
- 摘要
- 正文 HTML
- 封面图
- 阅读原文链接

正文结构：

```text
1. 今日一句话结论
2. 今日新增或本期变化
3. 本期聚焦
4. 关键证据
5. 趋势判断
6. 阅读原文
```

### 6.2 封面图

封面图用日报标题和小标题生成。

建议：

- 后端生成 SVG 风格矢量构图。
- 再转成 PNG/JPG 上传微信。
- 画面包含项目简称、标题、小标题、日期、域名 your-domain.example。
- 不使用复杂论文图，不放 DOI，不放过多文字。

注意：

微信公众号正文和素材接口通常更适合 PNG/JPG。即使本地使用 SVG 生成，也应最终转成 PNG/JPG。

### 6.3 微信发布状态

每期日报保存：

- `wechatDraftId`
- `wechatMediaId`
- `wechatThumbMediaId`
- `wechatStatus`
- `wechatError`
- `wechatUpdatedAt`

后台提供：

- 测试公众号连接
- 生成公众号草稿
- 查看草稿状态
- 重新生成封面
- 手动发布或复制内容

### Phase 4：质量回归

- 测试 3 类主题：
  - 高频方向：AI、材料、医学。
  - 中频方向：土木智能设计、3D 打印混凝土。
  - 低频方向：热载流子退化、DLC 阻感薄膜。
- 验证新增不足时不硬凑日报。
- 验证历史重复文献降权。
- 验证公众号草稿可读性。
